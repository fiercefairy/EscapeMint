# Release v0.28.0 - Closed Fund Reopen & Derivatives Fix

Released: 2026-01-20

## Overview

This release adds a confirmation dialog when adding entries to closed funds and fixes a critical bug in the derivatives liquidation price calculation.

## Bug Fixes

### Derivatives Liquidation Price Calculation
Fixed incorrect liquidation price calculation for BTC perpetual futures:

- **Root Cause**: The maintenance margin rate was set to 5% instead of the correct 20% for Coinbase BTC perpetuals
- **Symptom**: Liquidation prices were showing negative values (e.g., -$7,997) instead of realistic positive values (e.g., ~$7,500)
- **Fix**: Updated `DEFAULT_MAINTENANCE_MARGIN_RATE` from 0.05 to 0.20 and made it configurable per fund via the `maintenance_margin_rate` config field

### Files Changed
- `packages/engine/src/derivatives-calculations.ts` - Updated default rate and added parameter
- `packages/engine/src/types.ts` - Updated documentation for maintenance_margin_rate field
- `packages/server/src/routes/funds.ts` - Pass maintenance margin rate from config (6 locations)
- `packages/server/src/services/dashboard-cache.ts` - Pass maintenance margin rate from config
- `packages/server/src/utils/fund-metrics.ts` - Pass maintenance margin rate from config

## New Features

### Closed Fund Reopen Confirmation
When adding an entry to a fund with status "closed", users are now prompted to confirm reopening:

- **Confirmation Dialog**: "This fund is currently closed. Adding an entry will reopen it. Continue?"
- **Reopen & Save**: Confirms the entry and automatically sets fund status to "active"
- **Cancel**: Dismisses the dialog and stays on the entry form without saving
- **Toast Notification**: Shows "Entry added and fund reopened" on successful save

This improves the UX when revisiting closed positions or resuming trading in a previously closed fund.

## Technical Details

### Files Changed
- `packages/web/src/components/AddEntryModal.tsx` - Added confirmation dialog logic and state management
- `packages/web/src/pages/FundDetail.tsx` - Passes `fundStatus` prop to AddEntryModal

- Added `fundStatus` prop to `AddEntryModalProps` interface

- Created `saveEntry` helper for normal and reopen saves
