# Release v0.38.x

## v0.38.2

### Bug Fixes

- **Test platform detection**: `isTestPlatform()` changed from `endsWith('test')` to `startsWith('test')` to match global-setup/teardown cleanup logic, fixing orphaned test funds and platforms after E2E runs

## v0.38.1

### Changed

- **TWAP denominator for trading fund APY**: Replaced snapshot-based denominator fallback chain with Time-Weighted Average Position (TWAP) for trading fund APY calculations. Tracks `costBasis * days` during active periods so multi-cycle funds reflect actual average capital deployed rather than cumulative or current-snapshot values. Matches the existing TWAB pattern used by cash funds.

### Removed

- **`totalEverInvested` / `_previousCyclesGain`**: Removed from server fund-metrics (both only used for denominator selection, now replaced by TWAP). `previousCyclesGain` retained in FundDetail.tsx for total return display.

## v0.38.0

### Changed

- **Active days APY calculation**: APY now uses active days (time with capital deployed) instead of calendar days from first entry. Idle gaps after full liquidation no longer dilute APY.
- **Deprecate `start_date`**: `start_date` removed from fund config; start date is derived from first entry date. `start_date` field made optional for backward compatibility.
