# Release v0.38.x

### Changed

- **Active days APY calculation**: APY now uses active days (time with capital deployed) instead of calendar days from first entry. Idle gaps after full liquidation no longer dilute APY
- **TWAP denominator for trading fund APY**: Replaced snapshot-based denominator fallback chain with Time-Weighted Average Position (TWAP). Tracks `costBasis * days` during active periods so multi-cycle funds reflect actual average capital deployed. Matches the existing TWAB pattern used by cash funds
- **Deprecate `start_date`**: Removed from fund config; start date is derived from earliest entry date across trades and cashFlows with config fallback for backward compatibility
- **Fractional day precision**: Server and client both use fractional days for TWAP, TWAB, and activeDays calculations, eliminating APY discrepancies between fund detail page and aggregated metrics

### Bug Fixes

- **Test platform detection**: `isTestPlatform()` changed from `endsWith('test')` to `startsWith('test')` across all endpoints, matching global-setup/teardown cleanup logic
- **Cash fund aggregate metrics**: Pass cash flows to `computeFundMetrics` in aggregate endpoints so cash fund TWFS and daysActive are computed correctly
- **Out-of-order entry handling**: Sort entries chronologically in `computeFundFinalMetrics`; derive min/max dates via scan instead of assuming array order
- **Import readFund path**: Fixed `readFund` call to pass file path instead of bare fund ID
- **Cash interest backtest fallback**: `computeCashInterest` falls back to `asOfDate` instead of `new Date()`, preventing negative periods in historical calculations
- **TWAP same-day entries**: Same-date entries no longer artificially inflate TWAP denominator
- **Trading fund APY overflow**: Added upper-bound clamping to trading fund APY calculations, matching cash fund pattern. Prevents exponential blowup (e.g. 9.8e+36%) when TWAP denominator is very small relative to realized gains

### Removed

- **`totalEverInvested` / `_previousCyclesGain`**: Removed from server fund-metrics (replaced by TWAP). `previousCyclesGain` retained in FundDetail.tsx for total return display
