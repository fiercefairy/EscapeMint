# Release v0.11.0 - Derivatives Improvements & 100% Feature Coverage

Released: 2026-01-11

## Overview

This release brings significant improvements to the derivatives trading system with asset-agnostic calculations and unrealized P&L tracking, along with enhanced API logging and comprehensive test infrastructure improvements. We achieved **100% feature coverage** with 395 tests (223 unit + 172 E2E) all passing.

## 🎉 New Features

### Unrealized P&L for Derivatives
- **HOLD Entries Support**: Derivatives funds now support HOLD entries to track unrealized profit/loss
- **Mark-to-Market Tracking**: Record current position value without closing trades
- **Portfolio Integration**: Unrealized P&L properly reflected in portfolio charts and summaries

### Test Infrastructure Enhancements
- **Code Coverage Reporting**: Added comprehensive code coverage reporting with `npm run test:coverage-report`
- **Fast-Fail Testing**: New `npm run test:e2e:bail` script stops on first error for faster feedback during development
- **Coverage Reports in UI**: Coverage reports now accessible from Settings page

### Platform Management Improvements
- **Platform Cash Tracking API**: Enable/disable shared cash pools across platform funds
- **Platform Metrics API**: Aggregate fund values per platform
- **Platform Detail Page**: View all funds and metrics for a specific platform

## 🔧 Improvements

### API Error Logging
- **Emoji Indicators**: Replaced `[Error]` prefix with emoji-based log visibility
  - `⚠️` for 4xx client errors (400, 404, 422)
  - `❌` for 5xx server errors
- **Single-Line Format**: Known client errors now log as single lines without stack traces
- **Stack Traces for Unknowns**: Only truly unexpected 5xx errors include stack traces for debugging
- **Cleaner Logs**: Reduced log noise from expected validation and not-found errors

### Asset-Agnostic Derivatives
- **Flexible Contract Sizing**: Derivatives calculations no longer hardcoded to BTC
- **Dynamic Asset Support**: Engine now handles any asset's contract specifications (`notionalSize` replaces `btcSize`)
- **Improved Portability**: Easier to add support for new derivatives markets

### E2E Test Reliability
- **Fixed 45 Test Failures**: Resolved flaky tests across all test suites
- **Platform Management Tests**: Fixed API endpoint paths and test assertions
- **UI Workflow Tests**: Improved element selectors and wait conditions
- **Form Validation Tests**: Handle disabled button states correctly
- **Entry Management Tests**: Use deep links for reliable modal access
- **Import/Export Tests**: Updated tests to match current API contract

### Fund Store Improvements
- **Platform/Ticker Metadata**: Store metadata in JSON config files to handle multi-hyphen platform IDs correctly
- **Form Testability**: Added `name`, `id`, `role`, and `data-testid` attributes to all forms and modals

## 📋 Technical Details

### Error Logging Format

**Before:**
```
[Error] 404: Fund not found Error: Fund not found
    at createError (file:///...)
    at notFound (file:///...)
    ... (stack trace)
```

**After:**
```
⚠️ 404 NOT_FOUND: Fund not found
```

Server errors (5xx) still include full stack traces for debugging.

### Derivatives HOLD Entries

HOLD entries for derivatives work like stock/crypto funds:
- Record current market value without executing a trade
- Used for tracking unrealized gains/losses
- Properly calculated in portfolio aggregations

### Asset-Agnostic API Changes

The derivatives calculation functions now return `notionalSize` instead of `btcSize`:
- `formatPositionSummary()`: Returns `notionalSize` field
- `processTradeHistory()`: Returns `notionalSize` on each processed trade
- Supports any contract multiplier, not just BTC's 0.01
