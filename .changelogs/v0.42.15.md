# Release v0.42.15

Released: 2026-02-28

## Overview

Major import improvements (CashApp support, embedded backtest), cash fund normalization to HOLD entries, and a sweep of bug fixes across dashboard metrics, liquidation detection, and fund sizing.

## New Features

### CashApp CSV Importer
- Auto-detects CashApp CSV format by headers (`transaction_type`, `asset_type`, `net_amount`) and parses BUY/SELL/P2P transactions with correct cost basis using `net_amount` column

### Platform-Aware Import UI
- Import wizard now shows platform-specific text and passes the actual platform to the preview endpoint instead of hardcoding Robinhood

## Changes

### Backtest Page Embedded Inline
- The backtest page now renders directly within the app layout instead of using an iframe (dev) or redirecting to GitHub Pages (prod). All backtest engine code, components, and historical data are bundled into the web package.

### Cash Fund HOLD Normalization
- Cash entries are now normalized to HOLD action with signed amounts (positive for deposits, negative for withdrawals), replacing the legacy DEPOSIT/WITHDRAW actions
- Cash fund duplicate detection matches both HOLD and legacy DEPOSIT/WITHDRAW formats
- HOLD entries are supported in `entriesToCashFlows` and balance/TWFS calculations
- Platform check in export hint and cash flow derivation logic simplified and DRY'd up

## Bug Fixes

- **False liquidation detection on imported entries**: Fixed 10 locations where `value <= amount + 0.01` falsely triggered liquidation when `value` was 0 (unset). All checks now require `value > 0` before comparing
- **Pre-action equity values during import**: Apply endpoint now tracks running shares per fund and computes correct `value = running_shares * price` for each entry
- **Sell amounts in CashApp imports**: Uses `net_amount` (proceeds minus fees) instead of `amount + fee`
- **Dashboard P&L for accumulate-mode funds**: Aggregate endpoint now uses `computeFundFinalMetrics` for realized/unrealized gains and current value
- **Remaining false liquidation in `computeFundFinalMetrics`**: Added `value > 0` guard to the value-based liquidation check
- **Dashboard APY showing 0%**: Fixed `timeWeightedFundSize` calculation that was 0 for funds with `manage_cash: false` and no configured fund size
- **CashApp dividend import**: Added support for `Stock Dividends` transaction type in CashApp CSV parser
- **CashApp empty transaction type sales**: Handle sale rows where CashApp exports an empty `transaction_type` but includes "Sale of" in the notes
- **Dividend entries zeroing fund value**: Non-trade entries now use the last known asset price instead of defaulting to 0
- **Comma-formatted numbers in action form**: Values entered with commas (e.g., "1,000") now parsed correctly
- **Consistent TWFS day-bucketing**: Fixed time-weighted fund size to use consistent calendar day bucketing across engine and dashboard
- **Derivatives liquidation cycle detection**: Fixed derivatives liquidation detection in table display and historical import initialization
- **CashApp cash impact support**: Added date-aware last price initialization and platform display names
- **Recompute derived metrics after finalMetrics override**: Use calendar days for TWFS calculations
