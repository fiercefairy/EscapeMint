# Release v0.42.x

### Added

- **CashApp CSV importer**: Auto-detects CashApp CSV format by headers (`transaction_type`, `asset_type`, `net_amount`) and parses BUY/SELL/P2P transactions with correct cost basis using `net_amount` column
- **Platform-aware import UI**: Import wizard now shows platform-specific text and passes the actual platform to the preview endpoint instead of hardcoding Robinhood

### Changed

- **CLAUDE.md dev workflow docs**: Added development workflow section clarifying that UI is at port 5550 (not 5551) during dev, and that `npm run build` / `npm run build:web` should not be run during development to avoid downtime

### Fixed

- **False liquidation detection on imported entries**: Fixed 9 locations across the codebase where `value <= amount + 0.01` falsely triggered full-liquidation logic when `value` was 0 (unset). All checks now require `value > 0` before comparing
- **Pre-action equity values during import**: The apply endpoint now tracks running shares per fund and computes correct `value = running_shares * price` for each entry, fixing liquid P&L calculations that previously showed -100% on all entries
- **Sell amounts in CashApp imports**: Uses `net_amount` (proceeds minus fees) instead of `amount + fee`, which was inflating sell amounts
- **Dashboard P&L for accumulate-mode funds**: Aggregate endpoint now uses `computeFundFinalMetrics` for realized/unrealized gains and current value (not just APY), fixing funds with `accumulate: true` where the engine's `computeRealizedGains` returned $0 realized gains and pre-action entry values understated current value
- **Remaining false liquidation in `computeFundFinalMetrics`**: Added `value > 0` guard to the value-based liquidation check that was missed in the earlier 9-location fix
- **Dashboard APY showing 0%**: Fixed `timeWeightedFundSize` calculation in dashboard-cache that used `entry.fund_size ?? fund_size_usd`, which was 0 for funds with `manage_cash: false` and no configured fund size. Now tracks cumulative investment from BUY/SELL trades, matching the engine's approach
