# Release v0.37.x

## v0.37.5

### Changed

- **TWAP denominator for trading fund APY**: Replaced snapshot-based denominator fallback chain with Time-Weighted Average Position (TWAP) for trading fund APY calculations. Tracks `costBasis * days` during active periods so multi-cycle funds reflect actual average capital deployed rather than cumulative or current-snapshot values. Matches the existing TWAB pattern used by cash funds.

### Removed

- **`totalEverInvested` / `_previousCyclesGain`**: Removed from server fund-metrics (both only used for denominator selection, now replaced by TWAP). `previousCyclesGain` retained in FundDetail.tsx for total return display.

## v0.37.3

### New Features

- **Derivatives break-even & price scenario panel**: New panel on fund detail page showing position break-even price, net break-even (accounting for all deposits/withdrawals), and a price scenario calculator that shows projected equity, P&L, and margin at any target price
- **Position sizing calculator**: Added to the break-even panel to calculate how many contracts to add for a desired liquidation price, with real-time margin and equity projections

### Bug Fixes

- **Correct derivatives margin locked calculation**: For the final entry, `marginLocked` is recalculated at mark price as `abs(position) * contractMultiplier * effectiveAssetPrice * initialMarginRate`; for prior entries it uses FIFO queue margin sums
- **Fix liquidation price direction**: Liquidation price calculation accounts for position direction (long vs short)
- **Use derivEquity for current equity**: Break-even panel uses `derivEquity` instead of raw cash balance
- **Align initialMarginRate defaults**: Server callsites now use 0.25 (matching engine default) instead of 0.20
- **Guard against zero liquidation price**: Scraped liquidation price of 0 is treated as unknown rather than overriding calculated values

## v0.37.0

### Changed

- **Rename `cum*` variables to `sum*`**: Renamed all cumulative accumulator variables from `cum` prefix to `sum` prefix across 19 source files (e.g., `cumFees` → `sumFees`, `cumShares` → `sumShares`, `cumDividends` → `sumDividends`). Pure refactor with no behavioral or data changes.
