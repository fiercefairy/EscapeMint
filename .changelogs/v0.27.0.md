# Release v0.27.0 - Holiday Detection Fix & CI Improvements

Released: 2026-01-19

## Overview

This release fixes a timezone issue in the stock market holiday detection introduced in v0.26.0, and adds automatic build number incrementing to the CI pipeline.

## New Features

### Automatic Build Number Incrementing
- CI now automatically increments the patch version on every push to the dev branch
- Version format: `MAJOR.MINOR.BUILD` (e.g., `0.27.1`, `0.27.2`, etc.)
- Build numbers increment after tests pass, providing traceable version numbers for each dev commit
- Commits with `[skip ci]` in the message are excluded from triggering builds

## Bug Fixes

### Holiday Detection Timezone Fix
- Fixed timezone issue in holiday detection that caused incorrect results in non-UTC timezones
- The `isUSMarketHoliday` function was using local time methods (`.getFullYear()`, `.getMonth()`, `.getDate()`) on a UTC date
- Holiday calculations now consistently use UTC methods (`getUTCFullYear`, `getUTCMonth`, `getUTCDate`, etc.) to match the US Eastern Time normalization
- Affected holidays: All floating holidays (MLK Day, Presidents Day, Memorial Day, Labor Day, Thanksgiving) and Good Friday

## Technical Details

### Files Changed
- `packages/web/src/utils/format.ts` - Updated `isUSMarketHoliday` and `calculateEaster` to use UTC methods
- `.github/workflows/ci.yml` - Added bump-build job for automatic version incrementing on dev branch
