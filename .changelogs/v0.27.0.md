# Release v0.27.0 - Four Pillars Fund Categories

Released: 2026-01-19

## Overview

This release introduces the Four Pillars portfolio categorization system, enabling users to classify funds by investment philosophy: Liquidity, Yield, Store of Value (SoV), and Volatility. Margin is tracked separately as borrowing capacity. Includes support for multi-category "pie" funds that split allocations across multiple categories. Also includes fixes for timezone issues in holiday detection and CI improvements.

## New Features

### Four Pillars Fund Categories
Portfolio categorization system with 4 investment philosophy pillars, plus margin tracking:

- **Liquidity** - 24/7 cash access for spending and investing (Cash, Money Market)
- **Yield** - Optional stable high-yield instruments (STRC or similar stable dividend stocks)
- **Store of Value (SoV)** - Global store of value, hedge against fiat (BTC)
- **Volatility** - Capture whole market fluctuations (TQQQ, SPXL, UPRO)
- **Margin** - Tracked separately as borrowing capacity, not an allocation category

**Features:**
- New `FundCategory` type with 4 categories and configuration
- Category selector in Create/Edit fund modals (hidden for cash funds - always liquidity)
- Horizontal bar chart on dashboard showing portfolio allocation by category
- Margin capacity indicator (available vs borrowed) shown separately as overlay
- Category badge on fund cards (Liq, Yld, SoV, Vol)
- Auto-assign categories: cash→liquidity, derivatives→volatility, btc→sov, strc→yield
- Philosophy docs updated with "The Four Pillars of Portfolio Construction" section

### Multi-Category Pie Funds
Support for funds that span multiple categories (e.g., M1 Finance pies):

- New `CategoryAllocation` type with category and percentage fields
- "Multi-category (pie fund)" toggle in Create/Edit fund modals
- Configure arbitrary percentage allocations to different categories
- Validation that percentages sum to 100%
- Dashboard splits fund value across categories based on configured percentages
- Add/remove category allocations dynamically

### Automatic Build Number Incrementing
- CI now automatically increments the patch version on every push to the dev branch
- Version format: `MAJOR.MINOR.BUILD` (e.g., `0.27.1`, `0.27.2`, etc.)
- Build numbers increment after tests pass, providing traceable version numbers for each dev commit
- Commits with `[skip ci]` in the message are excluded from triggering builds

## Bug Fixes

### Holiday Detection Timezone Fix
- Fixed timezone issue in holiday detection that caused incorrect results in non-UTC timezones
- The `isUSMarketHoliday` function was using local time methods on a UTC date
- Holiday calculations now consistently use UTC methods to match US Eastern Time normalization
