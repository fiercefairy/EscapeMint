# Release v0.17.0 - Backtest Target Equity Fix & Mode-Specific Settings

Released: 2026-01-12

## Overview

This release fixes a critical bug in the backtest target equity calculation where the target was incorrectly decreasing when selling profits in accumulate mode. It also adds separate settings persistence for Accumulate vs Harvest modes, allowing users to maintain different configurations for each strategy.

## ✨ New Features

### Separate Settings for Accumulate vs Harvest Modes
- **Independent configurations**: Each mode now has its own localStorage settings that persist separately
- **Mode memory**: The app remembers which mode you last used and loads it on startup
- **Smart switching**: When toggling between modes, your settings for each mode are preserved
- **Mode-specific reset**: "Reset to Defaults" now only resets the currently selected mode

### New Default Settings for Accumulate Mode
- **Allocation**: SPXL 50%, TQQQ 25%, BTC 25% (more conservative than harvest)
- **Target APY**: 20% (vs 40% for harvest mode)
- All other settings remain the same between modes

## 🐛 Bug Fixes

### Target Equity Calculation in Accumulate Mode
- **Fixed**: Target equity now correctly continues growing after profit-taking sells in accumulate mode
- **Problem**: Previously, when selling profits in accumulate mode, the target equity would decrease proportionally - but the principal was still invested and should continue compounding
- **Solution**: Updated `computeExpectedTarget` and `computeStartInput` in the engine to check for `config.accumulate` and preserve `startInput` and `expectedGain` on partial sells

**Behavior by mode:**

| Mode | Partial Sell | Full Liquidation |
|------|-------------|-----------------|
| Accumulate | Target unchanged (profit extraction only) | Target resets to 0 |
| Harvest | Target reduced proportionally | Target resets to 0 |

### Data Path Resolution
- Fixed historical data fetch paths to use `import.meta.env.BASE_URL` for correct resolution in both dev and production
- Replaced external vite.svg favicon with inline emoji

### TypeScript Configuration
- Added missing `vite-env.d.ts` to fix `import.meta.env` type errors in pages app

## 🔧 Improvements

### Development Infrastructure
- **Port change**: Moved pages dev server from port 5552 to 5561 to avoid conflicts with other local applications
- **PM2 integration**: Added `escapemint-pages` as a managed PM2 app in `ecosystem.config.cjs`
- **Environment variable**: Pages port now configurable via `VITE_PAGES_PORT` environment variable
- **Centralized config**: All ports defined in single source of truth (`ecosystem.config.cjs`)

### GitHub Pages Deployment
- Deploy workflow now triggers from `dev` branch instead of `main`

## 📋 Technical Details

### Files Changed

**Engine (`packages/engine/src/expected-equity.ts`):**
- `computeExpectedTarget()` - Added accumulate mode check to preserve target on partial sells
- `computeStartInput()` - Added optional `config` parameter, respects accumulate mode

**Pages App (`pages/src/App.tsx`):**
- Separate localStorage keys: `escapemint-backtest-accumulate`, `escapemint-backtest-harvest`
- Mode preference key: `escapemint-backtest-last-mode`
- `getDefaultAccumulateConfig()` - New defaults for accumulate mode
- `getDefaultHarvestConfig()` - Existing defaults for harvest mode
- `handleConfigChange()` - Detects mode switches and loads appropriate config
- `handleReset()` - Only resets current mode's settings

**Configuration:**
- `ecosystem.config.cjs` - Added `PAGES: 5561` port, added `escapemint-pages` PM2 app
- `pages/vite.config.ts` - Changed port to 5561, disabled auto-open
- `packages/web/src/pages/Backtest.tsx` - Uses `VITE_PAGES_PORT` env var
- `pages/src/vite-env.d.ts` - Added Vite client types

**GitHub Actions:**
- `.github/workflows/pages.yml` - Deploy from `dev` branch

## 📦 Installation

```bash
git clone https://github.com/atomantic/EscapeMint.git
cd EscapeMint
npm run setup
npm run dev
```

If you have existing PM2 processes running, restart them to pick up the new pages app:

```bash
npm run dev:restart
```
