# Release v0.7.0 - Mobile & Margin Support

Released: 2026-01-10

## Overview

This release brings EscapeMint to mobile devices with a fully responsive UI, adds comprehensive margin expense tracking for cash accounts, and includes major improvements to charting and derivatives data collection.

## 🎉 New Features

### Mobile-Friendly UI
- **Responsive Dashboard**: Dashboard now adapts seamlessly to mobile screens with a compact list view for allocations, replacing complex pie charts with simple progress bars and colored indicators
- **Mobile-Optimized Forms**: All "Take Action" forms now stack vertically on phones, use 2-column grids on tablets, and maintain the full 4-column layout on desktop
- **Better Space Utilization**: Reduced padding and optimized layouts for small screens (iPhone and Android tested)

### Margin Expense Tracking
- **New MARGIN Action Type**: Record margin interest expenses on cash accounts that have margin borrowing enabled
- **Automatic Integration**: Margin expenses are tracked separately from deposits/withdrawals while preserving accurate cash flow calculations
- **Configuration Support**: Enable margin tracking per-fund with the `margin_enabled` configuration option

### Enhanced Charting
- **APY Chart Settings**: Added configurable Y-axis bounds with persistent settings stored in localStorage
  - Settings gear icon turns mint green when custom bounds are active
  - Customize min/max values to focus on specific performance ranges
- **Improved Gains Chart**: Three separate lines for Realized (green), Unrealized (amber), and Liquid (blue) gains
  - Per-fund breakdowns enable accurate platform filtering
  - Smart tooltip positioning that flips to the left near screen edges
  - Historical calculations respect fund closure dates

### Coinbase Integration Improvements
- **Browser Scraping**: Added CoinbaseScrapeButton for derivatives funds with improved modal handling
- **Smart Detection**: Automatically dismisses "Advanced Trade" popup before scraping
- **Accurate Leverage**: Fixed leverage formula to match Coinbase's calculation: `current_notional / margin_locked`
- **Better UX**: Keeps Coinbase tab open after scraping completes for easier verification

## 🔧 Improvements

### Calculation Accuracy
- Fixed liquid gain calculation to be `realized + unrealized` instead of `value - startInput`
- Corrected derivatives realized calculation to include all components (funding, interest, rebates, fees)
- Fixed unrealized gains for fully liquidated funds (properly sets to 0 after liquidation)
- Fixed historical calculations for closed funds (no longer forces currentValue=0 for past dates)

### Code Quality
- Simplified fully liquidated check to `costBasis === 0` (cleaner logic)
- Used `totalRealizedGains + totalUnrealizedGains` for better code clarity
- Fixed individual fund liquidAPY to include realized gains, not just paper gains
- Added JSDoc notes for brittle selectors in Coinbase import code

## 🗑️ Removed

- **Coinbase API Integration**: Removed unused API client and related components
  - Deleted ApiKeyModal, DerivativesDashboard, FundingTracker components
  - Deleted DerivativesFundDetail and PlatformSettings pages
  - Now rely on manual entry and browser scraping for derivatives data

```bash
cd EscapeMint

```
