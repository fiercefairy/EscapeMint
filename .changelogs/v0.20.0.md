# Release v0.20.0 - Data Entry Improvements

Released: 2026-01-14

## Overview

This release focuses on data entry quality with digit error detection for equity values and fixing floating point display issues in the backtest tool.

## New Features

### Auto-Apply Recommendations
- **Automatic action selection**: When entering equity value, the recommended action (BUY/SELL/HOLD) and amount are automatically filled in
- No need to click "Use" button - form updates as you type (after brief debounce)
- Manual "Use" button still available if recommendation changes

### Entry Form Wizard
- **Guided data entry**: When opening the Take Action form, an animated indicator guides users through the primary fields
- **Step 1**: Equity field is highlighted with pulsing arrow showing "Update first"
- **Step 2**: After equity is updated, Margin Available field is highlighted (if margin enabled)
- Visual ring and border highlight make it clear which field needs attention

### M1 Auto-Borrow Margin
- **Automatic margin tracking**: For M1 platform, when a BUY exceeds available cash, the "Margin Borrowed" field is automatically updated to reflect the expected new total
- Calculates: initial margin borrowed + shortfall amount
- Resets to initial value if shortfall is reduced/eliminated
- Shows purple note: "Auto-adjusted: +$X for shortfall (M1 auto-borrows from margin)"

### Equity Input Validation
- **Digit error detection**: When entering equity values in the Take Action form, the system now detects likely fat-finger errors where a digit was accidentally added or removed
- Compares new value against prior entry's equity - if it's approximately 10x higher or lower, triggers a warning
- **Warning banner**: Persistent amber warning banner appears at top of modal when digit error detected
- **Toast on blur**: Additional toast notification when leaving the equity field
- Allows natural transitions (e.g., 999 -> 1000) by checking if the actual percentage change is reasonable
- Warning displays both values: "Prior equity: $12,052.00 → You entered: $120,052.00"

## Bug Fixes

### Captured Profit Chart Fix (Accumulate Mode)
- **Fixed extracted value going negative**: In accumulate mode, the chart's "Extracted" line would incorrectly drop when total sells exceeded cost basis
- Root cause: `dollarLiquidated` check was triggering false "full liquidation" in accumulate mode
- Fix: Dollar-based liquidation check now only applies in harvest mode, not accumulate mode

### Backtest Slider Precision
- **Fixed floating point display errors**: Slider values like Target APY now display clean integers (e.g., `29%`) instead of floating point artifacts (e.g., `28.999999999999996%`)
- Applies to both `SliderField` in BacktestView and `NumberSlider` in ScenarioConfig
- Values are rounded based on step precision (integer for step=1, one decimal for step=0.5)
