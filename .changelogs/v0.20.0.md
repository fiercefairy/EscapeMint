# Release v0.20.0 - Data Entry Improvements

Released: 2026-01-14

## Overview

This release focuses on data entry quality with digit error detection for equity values and fixing floating point display issues in the backtest tool.

## New Features

### Equity Input Validation
- **Digit error detection**: When entering equity values in the Take Action form, the system now detects likely fat-finger errors where a digit was accidentally added or removed
- Compares new value against prior entry's equity - if it's approximately 10x higher or lower, shows a warning toast
- Allows natural transitions (e.g., 999 -> 1000) by checking if the actual percentage change is reasonable
- Warning displays both values to help user identify the mistake (e.g., "Possible extra digit? Prior equity was $12,052.00, you entered $120,052.00")

## Bug Fixes

### Backtest Slider Precision
- **Fixed floating point display errors**: Slider values like Target APY now display clean integers (e.g., `29%`) instead of floating point artifacts (e.g., `28.999999999999996%`)
- Applies to both `SliderField` in BacktestView and `NumberSlider` in ScenarioConfig
- Values are rounded based on step precision (integer for step=1, one decimal for step=0.5)
