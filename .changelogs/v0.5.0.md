# Release v0.5.0 - Code Quality & Browser Scraping

Released: 2026-01-05

## Overview

This release focuses on code quality improvements, reducing over 1,000 lines of duplicate code through DRY refactoring, and introduces browser-based scraping for M1 Finance cash imports with real-time progress tracking via Server-Sent Events.

## 🎉 New Features

### Browser Scraping with Real-Time Progress
- **M1 Finance Cash Account Scraper**: Import cash transactions directly from M1 Finance website
- **Server-Sent Events (SSE)**: Real-time progress updates during scraping operations
- **Pagination Support**: Handles large transaction histories with incremental archiving
- **Duplicate Detection**: Automatically skips existing transactions

### Shared Cash Fund for Recommendations
- Trading funds with `manage_cash=false` now use platform's shared cash fund for recommendations
- Automatic routing to platform cash fund (e.g., `m1-cash`) for M1 Finance imports
- Better integration between trading and cash fund management

### API Utilities
- Created centralized `fetchJson`, `postJson`, `putJson`, `deleteResource` helpers
- Reduced ~300 lines of duplicate fetch code across web package
- Consistent error handling and response parsing
- Created `createEventStream` helper for SSE handling

### Server Calculation Utilities
- `calculateStartInputWithLiquidation`: Centralized start input calculation
- `sortEntriesByDate`: Consistent entry sorting
- Shared utilities reduce code duplication in API routes

## 🐛 Bug Fixes

### Calculation Accuracy
- **SELL Handling**: Fixed `computeExpectedTarget` to proportionally reduce expected gain on sells
- **APY Denominator**: Fixed `computeClosedFundMetrics` to use `totalInvested` for correct ROI calculation
- **Config Implementation**: Properly implemented `dividend_reinvest`, `interest_reinvest`, and `expense_from_fund` options

### TypeScript & Build
- Fixed TypeScript declaration emit for Express routers
- Added explicit type annotations for better type safety in routes
- Improved build reliability for CI/CD

## 🔧 Improvements

### DRY/YAGNI Audit
- Removed ~1,000 lines of duplicate/unused code
- Deleted unused SubFund feature (5 files, ~400 lines)
- Removed unused `calculateAggregateMetrics` legacy function
- Consolidated platform file utilities (`readPlatformsData`, `writePlatformsData`)

### Code Organization
- Added `HOLD` action type to Recommendation interface for clearer intent
- Better separation of concerns between packages
- Improved code maintainability and testability

### E2E Tests
- Fixed tests to properly respect `manage_cash=true` on trading funds
- Allow DEPOSIT/WITHDRAW actions for funds with `manage_cash=true`
- More realistic test scenarios
