# Release v0.8.0 - Comprehensive Test Coverage

Released: 2026-01-10

## Overview

This release establishes a robust testing foundation for EscapeMint with **85% overall feature coverage** and **100% coverage of critical features**. The new test infrastructure includes automated coverage reporting, comprehensive end-to-end tests, and a systematic feature matrix covering 114 features across 19 categories.

## 🎉 New Features

### Test Coverage System
- **Coverage Matrix**: Comprehensive tracking of 114 features across 19 categories
  - 100% critical features tested
  - 93% high-priority features tested
  - 85% overall feature coverage
- **HTML Coverage Report**: Visual coverage report generator accessible via `npm run test:coverage-report`
- **Actionable Tracking**: Clear TODO comments for untested features with priority context

### End-to-End Test Suites

#### Derivatives Trading Tests (`derivatives-funds.spec.ts`)
- 20+ tests covering perpetual futures lifecycle
- Position tracking, funding payments, interest accrual
- Leverage calculations, liquidation scenarios
- Realized vs unrealized P&L tracking

#### Comprehensive UI Workflows (`ui-workflows.spec.ts`)
- 25+ tests covering complete user journeys
- Fund creation, editing, deletion
- Entry management (add, edit, delete)
- Chart interactions and filtering
- Platform and fund navigation
- Settings and configuration
- Import/export workflows

#### Platform Management (`platform-management.spec.ts`)
- Platform CRUD operations
- Platform-specific fund handling
- Validation and error cases

#### Import/Export Functionality (`import-export.spec.ts`)
- TSV export with proper formatting
- Import with validation
- Round-trip integrity testing
- Error handling for invalid data

#### Cash Fund Management (`cash-funds.spec.ts`)
- Complete cash fund lifecycle
- Interest calculations
- Expense tracking
- Balance validation

### Test Infrastructure Improvements
- **Isolated Test Data**: Tests use "test" platform to avoid contaminating real data
- **API Test Support**: Endpoints support `?include_test=true` parameter
- **Single Worker**: Playwright runs with single worker to prevent data conflicts
- **Test Fixtures**: Comprehensive test data generator with realistic scenarios
- **Dynamic Test Dates**: Tests use relative dates (1 year ahead) to prevent date-related failures

## 🔧 Code Quality Improvements

### Testing Best Practices
- Removed unused imports and variables across test files
- Added proper assertions to all tests (no silent passes)
- Fixed conditional test patterns that could hide failures
- Added negative test cases for better validation coverage
- Improved error message verification with explicit checks
- Added `test.skip()` for optional features instead of conditional logic

### Test Helpers & Utilities
- `uniqueTicker()`: Generates unique test tickers using `crypto.randomUUID()`
- `uniquePlatformId()`: Creates unique platform IDs for test isolation
- `testDate()`: Helper for relative date calculations
- `escapeHtml()`: XSS prevention in report generator
- `HYDRATION_DELAY_MS`: Configurable delay for CI environments

### Documentation
- Clarified funding payment calculations in derivatives tests
- Updated README with current setup instructions
- Updated PLAN.md with coverage audit results
- Added TODO comments for high-priority untested features

## 🗑️ Cleanup

- **Removed `data.example/` directory**: No longer needed with test data generator
- Simplified `setup:data` script to just create `data/funds` directory
- Removed stale references from documentation

## 🚀 Running Tests

```bash
# Run all tests
npm run test

# Run end-to-end tests
npm run test:e2e

# View test coverage report
npm run test:coverage-report

# Run E2E with UI
npm run test:e2e:ui
```
