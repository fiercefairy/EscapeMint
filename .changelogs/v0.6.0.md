# Release v0.6.0 - Derivatives Trading & Platform Consolidation

Released: 2026-01-09

## Overview

This major release introduces comprehensive support for perpetual futures trading with Coinbase integration, centralized fund type configuration, browser-based transaction scraping, iCloud backups, and extensive documentation improvements.

## 🎉 New Features

### Derivatives (Perpetual Futures) Trading
- **New Fund Type**: `derivatives` for tracking perpetual futures positions
- **FIFO Cost Basis Tracking**: Proper lot tracking for tax purposes
- **Comprehensive Calculations**:
  - Position size, average entry price, and contracts held
  - Unrealized and realized P&L with historical snapshots
  - Margin tracking: locked, available, and maintenance margin
  - Liquidation price calculation based on equity and leverage
  - Funding payments, interest, rebates, and fees
- **Safe Order Ladder**: Suggests limit orders that keep equity positive at $0 BTC
- **Derivatives-Specific UI**: Custom charts for price/liquidation, capital/leverage, and P&L tracking

### Coinbase Integration
- **Secure API Key Management**: Stored in macOS Keychain (never in files or logs)
- **JWT Authentication**: Fresh ES256 tokens for each request with 2-min expiry
- **Read-Only Access**: All operations are GET-only, no trade execution
- **Real-Time Data**: Positions, portfolio summary, fills, funding payments, and current prices
- **Browser Scraping**: Alternative to API with transaction page parsing
- **Archive Management**: Manual and bulk import for funding payments and rewards

### Fund Type Configuration System
- **Centralized Config**: Created `packages/engine/src/fund-type-config.ts`
  - `FUND_TYPE_DEFAULTS`: Default configuration values per fund type
  - `FUND_TYPE_FEATURES`: Feature flags (trading, recommendations, dividends, etc.)
  - `ALLOWED_ACTIONS`: Valid entry actions per fund type
  - Helper functions: `isCashFund()`, `isDerivativesFund()`, `isTradingFund()`, `getFundTypeFeatures()`
- **Reduced Duplication**: Refactored 11 files to use centralized config
- **Easier Maintenance**: Adding new fund types now requires changes to 1-2 files instead of 10+

### Browser-Based Scraping
- **Coinbase Transactions**: Scrape trades, funding, rewards, and rebates from transaction page
- **Early Exit Optimization**: Stops when consecutive existing transactions found (50 for Robinhood/Coinbase, 30 for M1)
- **SSE Streaming**: Real-time progress updates during scraping
- **Archive Storage**: Scraped data stored in platform-specific JSON archives
- **CoinbaseScrapeButton**: UI component with progress visualization

### iCloud & Local Backups
- **Automatic Backups**: Create timestamped backups of all fund data
- **iCloud Support**: Default to iCloud Drive on macOS
- **Backup Browser**: View and manage recent backups in Settings
- **One-Click Restore**: Restore from any backup with confirmation dialog
- **Comprehensive Snapshots**: Includes funds, platforms, totals, and scrape archives

### Test Data Generator
- **Historical DCA Simulation**: Generate realistic test funds using 5 years of real price data
- **Real Market Data**: BTCUSD, TQQQ, and SPXL historical prices bundled
- **Weekly DCA**: Simulates $100/week investments every Wednesday
- **Test Funds Mode**: Toggle between test and real data in Settings
- **Demo Platform**: All test funds on single `test` platform with automatic cleanup
