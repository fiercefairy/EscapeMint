# Release v0.10.0 - Documentation & Configuration Improvements

Released: 2026-01-11

## Overview

This release focuses on improving the development workflow and documentation with a new structured changelog system, simplified configuration management, and integrated test report viewing. The project now uses a dedicated `.changelogs/` directory for release notes, has removed unnecessary `.env` files in favor of centralized PM2 configuration, and hosts Playwright and coverage reports directly in the web UI for easy access.

## 🎉 New Features

### Structured Changelog System
- **Dedicated Changelog Directory**: Created `.changelogs/` directory for version-specific release notes
- **Rich Release Notes**: Each version now has detailed markdown changelog with sections for features, fixes, improvements, and removals
- **Automated Release Workflow**: GitHub Actions release workflow automatically uses changelog files when available
- **Changelog Documentation**: Added comprehensive `.changelogs/README.md` with format guidelines and best practices
- **Fallback Generation**: Release workflow falls back to commit-based changelog if no file exists

### Test Reports in Web UI
- **Hosted Playwright Reports**: E2E test reports now served from `packages/web/public/playwright-report/`
- **Hosted Coverage Reports**: Feature coverage reports now served from `packages/web/public/coverage-report/`
- **Settings Page Links**: Added "Test Reports" section to Settings page with links to view both reports
- **External Link Icons**: Report buttons open in new tabs with external link indicators
- **Usage Instructions**: Added helpful text showing which npm commands generate the reports

## 🔧 Improvements

### Backup Management UI Overhaul
- **Reorganized Settings Page**: iCloud Backup now prominently featured as the first card in a 3-column grid layout
- **Delete Backup Functionality**: Added delete button with confirmation dialog for individual backups
- **Scrollable Backup List**: Full backup list is now scrollable with max height, showing count in header
- **Enhanced UI Aesthetics**:
  - Full-width Create Backup button with icon and loading spinner
  - Document icons next to each backup entry
  - Hover effects and better visual hierarchy
  - Improved button styling with proper disabled states
  - Monospace font for backup dates for better readability
  - Darker background for backup list with dividers between items
- **Backend API Enhancements**:
  - Added `DELETE /api/v1/backup/:filename` endpoint for deleting backups
  - Added `GET /api/v1/backup/download/:filename` endpoint for downloading backups
  - Added `POST /api/v1/backup/upload` endpoint for uploading backup files
  - Improved error handling in delete operations
  - Fixed route ordering to prevent conflicts between parameterized routes
- **Streamlined Layout**: Removed non-actionable "Data Storage" card from Settings page

### Configuration Management
- **Removed .env Files**: Eliminated `.env` and `.env.example` files (never loaded by code anyway)
- **Centralized Configuration**: All environment variables now set exclusively in `ecosystem.config.cjs`
- **Simplified Setup**: No need to manage separate `.env` files - PM2 handles all config
- **Smart Defaults**: All `process.env` usage has sensible fallbacks (e.g., `DATA_DIR ?? './data'`)

### Documentation Updates
- **CLAUDE.md Enhancement**: Added changelog creation instructions to development workflow
- **Release Process**: Documented how to create version-specific changelogs before releasing
- **Format Guidelines**: Clear structure and examples for writing release notes

### Project Metadata
- **Updated LICENSE**: Copyright year updated to 2025 with proper attribution
- **Package.json Author**: Added author field with GitHub handles
- **Cleaner .gitignore**: Removed obsolete `.env` patterns

## 🗑️ Removed

### Deprecated Files
- **CHANGELOG.md**: Removed root changelog file (replaced by `.changelogs/` directory)
- **.env & .env.example**: Removed environment files (config now in `ecosystem.config.cjs` only)
- **test/ directory**: Removed defunct test fixtures directory (using `e2e/` for all tests now)
- **.gitignore entries**: Removed `.env` patterns, added legacy `test/` pattern

## 📋 Technical Details

### Environment Variables
All environment variables are now set in `ecosystem.config.cjs`:
- `DATA_DIR`: Data directory path (defaults to `./data`)
- `PORT`: API server port (defaults to `5551`)
- `CDP_PORT`: Chrome DevTools Protocol port (defaults to `5549`)
- `NODE_ENV`: Node environment (`development` or `production`)
- `VITE_PORT`: Web UI port (defaults to `5550`)

No `dotenv` package is used - PM2 sets all variables directly.

### Changelog Workflow
1. During development: Track changes as you work
2. Before release: Create `.changelogs/v{version}.md` file
3. Follow format in `.changelogs/README.md`
4. Commit changelog with release prep
5. GitHub Actions uses it automatically for release notes

### Test Reports
Test reports are now hosted in the web app's public directory:
- **Playwright reports**: Run `npm run test:e2e` to generate at `packages/web/public/playwright-report/`
- **Coverage reports**: Run `npm run test:coverage-report` to generate at `packages/web/public/coverage-report/`
- **Access**: Navigate to Settings page and click "E2E Test Report" or "Coverage Report" buttons
- **Vite serving**: Reports are automatically served by Vite at `/playwright-report/` and `/coverage-report/`

Reports open in new browser tabs for easy viewing alongside the application.
